- 현재 상태: 복원 완료
- 이미지 파일명: assets/palgle-screenshot.png
- 한 줄 요약: Workpress 데이터를 Markdown로 변환하여 jekyll을 이용한 정적 사이트로 복원했습니다.
- 복구 노트 내용:
  1. DB와 워드프레스에서 데이터 추출
  - 워드프레스에 엑스트라 플러그인을 이용하여 데이터를 json으로 추출했습니다.
  - 엑스트라 플러그인을 이용한 방법이 가장 안전하고, 정확한 방법입니다. (단, 필드가 추가되었을 경우 플러그인도 업데이트를 해야함)
  - 워드프레스 DB와 직접 연동하여 데이터를 추출하는 방법도 있습니다. 이 방법은 가장 정확하고 빠른 방법입니다.
  - 워드프레스 DB와 직접 연동하는 방법의 가장 큰 장점은 플러그인을 사용하지 않아도 된다는 것입니다.
  2. 데이터를 Markdown 포멧으로 변경
  - 워드프레스의 데이터를 마크다운으로 변경해야합니다. 워드프레스의 데이터는 mysql에 저장되어 있습니다.
  - 워드프레스 플러그인을 이용하여 데이터를 마크다운으로 변경하는 방법이 가장 안전하고, 정확한 방법입니다. (단, 필드가 추가되었을 경우 플러그인도 업데이트를 해야함)
  - 워드프레스 DB와 직접 연동하여 데이터를 마크다운으로 변경하는 방법도 있습니다. 이 방법은 가장 정확하고 빠른 방법입니다.
  - 우리는 정합성을 위해서 워드프레스 플러그인을 사용하지 않고 자체 프로그램을 제작하여 데이터를 마크다운으로 변경했습니다.
  3. Jekyll 설치
  - Jekyll을 설치하고 Github Actions를 이용하여 Github Pages에 자동 배포하는 환경을 구축했습니다.
  - 이를 통해 무료로 블로그를 운영할 수 있게 되었습니다.
  - Github pages는 정적 사이트만 지원하므로 동적인 기능은 제거되었습니다.
  4. Markdown Formatter
  - Node.js 코드를 통해 모든 글들이 Markdown으로 포매팅 되었습니다.
  5. 링크와 이미지 복구
  - 1000여개의 글에서 링크가 깨지는지를 확인했고, 모든 링크를 복구했습니다.
  - 이미 없어진 링크는 링크 코드를 삭제했습니다.
  - 깨진 이미지는 텍스트로 변경하거나 이미지 자체를 삭제했습니다.
  - 이미지는 Github 저장소에 업로드하여 Github CDN을 통해 제공했습니다.
