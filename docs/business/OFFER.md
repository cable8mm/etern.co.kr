# ETERNOps

Service Packages

운영 중인 웹·모바일 서비스를 안전하게 다음 상태(Next State)로 이동시킵니다.

우리는 새로운 서비스를 무작정 만드는 것보다,
이미 성공적으로 운영되고 있는 서비스를 더 안정적이고 지속 가능한 상태로 만드는 일에 집중합니다.

---

# 1. Service Assessment

## Technical Assessment

현재 시스템 상태를 객관적으로 파악하고
가장 위험한 문제부터 우선순위를 정리합니다.

많은 기업은 현재 서비스가 어떤 상태인지 정확히 알지 못한 채 운영하고 있습니다.

개발자가 퇴사했거나,
문서가 없거나,
서비스가 오래되었거나,
외주 개발사가 구축한 시스템을 인수받은 경우가 대표적입니다.

Technical Assessment는 이러한 불확실성을 제거하기 위한 첫 단계입니다.

---

## 이런 경우에 적합합니다

- 개발자가 인수인계 없이 퇴사했다
- 현재 시스템 구조를 아는 사람이 없다
- 외주 개발사가 만든 시스템을 검증하고 싶다
- 서비스가 느리거나 불안정한데 원인을 모른다
- 이전(Migration) 또는 현대화(Modernization)를 검토하고 있다
- 기술 부채가 어느 정도인지 파악하고 싶다

---

## 제공 범위

### Web & Mobile Analysis

- 웹 서비스 구조 분석
- 모바일 앱 구조 분석
- 빌드 및 배포 환경 분석

### Infrastructure Review

- AWS 환경 점검
- Docker 환경 점검
- 서버 운영 구조 검토
- 배포 프로세스 검토

### Technical Debt Assessment

- 레거시 코드 분석
- 잠재 장애 요소 식별
- 유지보수 난이도 평가

### Priority Roadmap

- 우선 개선 항목 정의
- 위험도 분석
- 단계별 개선 로드맵 제안

---

## 산출물

### Architecture Assessment Report

포함 내용:

- 현재 시스템 구조
- 주요 리스크 요인
- 기술 부채 현황
- 개선 우선순위
- 추천 실행 계획

---

## 기간

3~5 영업일

---

## 비용

300만 원 ~ (단일 서비스/플랫폼 기준, VAT 별도)

# 2. Modernization & Migration

## Situation-Based Solutions

운영 중인 서비스를
보다 안정적이고 유지보수 가능한 상태로 개선합니다.

모든 프로젝트는 현재 서비스의 운영 연속성과 데이터 안전성을 최우선으로 고려하여 진행합니다.

---

## Type A

### Service Modernization

돌아가고는 있지만
불안정하거나 관리가 어려운 서비스를 개선합니다.

---

### Web Modernization

예시

- PHP 업그레이드
- 레거시 프레임워크 개선
- WordPress 현대화
- 프론트엔드 환경 정비
- 성능 개선

---

### Mobile Modernization

예시

- Android SDK 업그레이드
- iOS SDK 업그레이드
- Flutter 환경 개선
- 빌드 파이프라인 개선
- 기술 부채 제거

---

### Infrastructure Modernization

예시

- Docker 도입
- CI/CD 구축
- AWS 구조 개선
- 비용 최적화
- 운영 자동화

---

## Type B

### Commerce & Platform Migration

서비스와 데이터를
안전하게 새로운 플랫폼으로 이전합니다.

---

### 대표 사례

- Cafe24 이전
- 고도몰 이전
- 영카트 이전
- 자체 쇼핑몰 이전
- CMS 이전
- 서비스 플랫폼 이전

---

### 핵심 원칙

- 데이터 손실 최소화
- 다운타임 최소화
- 운영 연속성 확보
- 검증 가능한 이전 절차 수행

---

### 주요 대상 데이터

- 회원 정보
- 주문 데이터
- 포인트 데이터
- 상품 데이터
- 콘텐츠 데이터
- 운영 이력 데이터

---

## 프로젝트 산출물

### Updated Production Environment

- 개선된 소스코드
- 개선된 운영 환경
- 개선된 배포 환경

---

### System Runbook

포함 내용

- 새로운 개발자나 외주사가 와도 즉시 가동 가능한 수준의 인수인계 매뉴얼
- 표준 운영 및 배포 절차
- 단계별 장애 대응 및 트러블슈팅 가이드

---

### Stabilization Support

- 오픈 직후 모니터링
- 주요 장애 대응 지원
- 안정화 지원

---

## 기간

일반적으로 2주 ~ 8주

---

## 비용

일반적으로 1,000만 원 ~

# 3. Technical Advisory

서비스를 운영하면서 발생하는 기술적 의사결정을 지원합니다.

개발 인력을 대체하는 서비스가 아니라,
기술 리스크를 줄이고 올바른 방향을 결정하기 위한 자문 서비스입니다.

---

---

- 기술 방향 검토

---

### Incident Support

- 장애 원인 분석
- 대응 방향 제안
- 기술적 의사결정 지원

# Project Process

모든 프로젝트는 다음 절차를 따릅니다.

---

## 1. Consultation

현재 가장 중요한 문제를 정의합니다.

---

## 2. Assessment

현재 시스템 상태를 정밀 진단하여 근본적인 문제와 리스크를 파악합니다.
(Technical Assessment 진행)

---

## 3. Planning

안전한 개선 또는 이전 계획을 수립합니다.

내부적으로 Migration Blueprint 방법론을 활용합니다.

---

## 4. Execution

핵심 기술 작업을 수행합니다.

- 현대화
- 이전
- 안정화
- 성능 개선

---

## 5. Stabilization

오픈 이후 안정화를 지원합니다.
